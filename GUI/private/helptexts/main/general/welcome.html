Welcome to the trEPR toolbox GUI.

New here? Or interested in a more consistent introduction in the concepts of the trEPR toolbox and mainly its GUI? This is the place to start...

If in a hurry

Suppose you don't want to read long introductions, but just start processing your data recorded right now: Read the section "GUI in 2 minutes". Besides that, please note that each panel and each additional GUI window comes with its own in-built help.

All those with more time and interest in a slightly more detailed introduction, keep reading.

History

What you have in hands is the result of now about eight years of more or less constant development. It all started out in 2005 with the need of having a set of scripts to process trEPR spectra. First being Matlab scripts, later functions, at some point the need for an easy-to-use graphical interface appeared...

The question why a GUI was introduced is covered in a separate section. The good news for all those more interested in the command line: All functionality can still be accessed via the command line, and in addition, starting with version 0.3.19 in early 2013, a command line (CMD) found its way into the GUI, laying out the path for a complete scripting of the GUI.

Key concepts

The key aspects of developing the GUI (and completely redesigning the toolbox) can be summed up with three keywords:

  • Modularity
  • Robustness
  • Usability

Besides that, there are a few more aspects worth to be mentioned explicitly, particularly as they are not as obvious, but thought to be at least as important if it comes to processing and handling of scientific data:

  • Metadata
  • History of processing
  • Documentation

These aspects are described in more detail on the key concepts page. The idea behind is to provide the user with a complete workflow for processing, analysing and displaying/representing the trEPR data recorded, while keeping a complete record and overview of every single step. Much of that is still work in progress, whereas the toolbox is since years in use in a productive (scientific) environment.

Development

As mentioned already, the toolbox is under constant development and as of now (May 2013) not available in a "stable" version ready for a productive environment with people not too much interested in debugging software and working around bugs.

A up-to-date list of new features gives an overview of the changes from version to version. There is as well a list of known bugs and a bit of documentation for how to file a (useful) bug report.
