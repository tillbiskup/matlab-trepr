Data processing

A set of standard data (pre)processing functions are available from this panel. It is divided into three pages.

A short description of all the controls of the panel's first page.

Select dataset

A list of currently visible datasets.

Standard corrections

The two corrections routinely performed on TREPR datasets:

  • pretrigger offset compensation (POC)
  • background correction (BGC)

The pretrigger offset compensation subtracts the average of the pretrigger part of every time trace from the whole time trace and therefore corrects for DC offsets in the data that are frequently encountered.

The background subtraction is an "fsc2 style" basic background correction that takes the first few time traces, averages over those, and subtracts the average from every time trace of the dataset, therefore correcting for the laser-induced background that is visible in off-resonant signal parts.

Whereas the former can be applied without damage or side effect to every dataset, the same is not necessarily true for the background subtraction.

Advanced corrections (dialogues)

For more advanced "corrections", two separate GUI windows are available:

  • baseline correction (BLC)
  • accumulation (ACC)

Both GUI windows come with their separate online help.

Scaling

Sometimes you want to scale the dataset without using the slider, e.g., if the slider range is not sufficient.

Please note: This feature does scale directly the intensity of your dataset, and this behaviour may change in the future.

Frequency correction

Often, different datasets are recorded at slightly different microwave frequencies. To directly compare those datasets, one needs to correct them for the same frequency.

Here you can set a microwave frequency (in GHz, as shown) an apply this to either all or only the current dataset. Alternatively, you can use the "fc" command from the GUI CMD.

Smoothing (running average filter)

To reduce the noise level, it is often convenient to smooth data with a running average filter with a given filter window.

Currently, three filters are available: boxcar, binomial, and Savitzky-Golay. Whereas boxcar shall normally not be used due to it strongly distorting your signals, the other two have less impact on your overall signal.

Be aware of that filtering with a window too large compared to the spectral features can easily lead to distortions of the spectra (comparable to "overmodulating" in cw-EPR). Therefore, use with care.
