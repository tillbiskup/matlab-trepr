disp

Set various display-specific parameters.

Description

The disp command is used to change display-related settings of the GUI, starting with the display modes (x,y,z/2D), continuing with settings such as legend and grid.

Syntax

disp

disp <parameter>

disp <parameter> <value>

disp <parameter> <value> <value>

Parameters

The option <parameter> following the disp command can currently take the following values:

x|y|z
Each of these parameters will set one of the corresponding display modes, named "1D along x", "1D along y" and "2D plot".
Currently, these parameters do not take any additional value.
grid
Set grid-related parameters.
Has to be followed by one of the values described in more detail in the following. Values currently supported are:
x
Toggle display of grid lines for x values.
y
Toggle display of grid lines for x values
minor
Toggle display of minor grid lines.
zero
Toggle display of zero line.
Each of these values can be followed by an additional parameter that can be either on or off. This allows programmatic and unambiguous setting of the respective values.
legend
Set legend-related parameters.
Has to be followed by one of the values described in more detail in the following. Values currently supported are:
none
Do not display any legend.
auto|best
Let Matlab™ decide where to place the legend.
nw|northwest
Locate legend in the upper left corner of the figure.
ne|northeast
Locate legend in the upper right corner of the figure.
sw|southwest
Locate legend in the lower left corner of the figure.
se|southeast
Locate legend in the lower right corner of the figure.
box
Toggle between displaying a box around the legend.
This value can alternatively be followed by an additional parameter that can take the values on and off, used to programmatically set the box display of the legend.
onlyactive
Toggle displaying only the active dataset.
Can be followed by an additional parameter that can be either on or off. This allows programmatic and unambiguous setting of the respective value.
stdev|std
Toggle displaying of the standard deviation (std|stdev) in combination with averaged data.
Can be followed by an additional parameter that can be either on or off. This allows programmatic and unambiguous setting of the respective value.
norm|normalise|normalize|normalisation|normalization
Set normalisation parameters.
Has to be followed by one of the values described in more detail in the following. Values currently supported are:
none|off
Do not normalise the display.
pkpk|pk-pk|peaktopeak|pk2pk|peak2peak
Set peak to peak ratio to one.
area
Divide by total area under curve.
min
Set minimum to one.
min
Set maximum to one.
limit|limits
Set limits of the axes in all three dimensions.
Has to be followed by one of the values described in more detail in the following. Values currently supported are:
auto
Set axes automatically to fit the data displayed.
Can be followed by an additional parameter that can be either on or off. This allows programmatic and unambiguous setting of the respective value.
x|y|z
Set axes in x, y, and z direction explicitly.
Has to be followed by two numeric values for the minimum and maximum.

Further Information

Eventually, every display-related setting will be accessible via this or similar commands from the command line of the trEPR GUI.
