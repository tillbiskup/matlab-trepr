export

Export either the current axis as graphics file or datasets.

Description

The export command is used to export either the current axis or datasets. The first option determines what to export, other additional options are used to specify the export further.

Syntax

export fig[ure]|axis|axes [format=<format>] [type=<filetype>] [caption=yes|no|y|n|true|false] [<filename>]

export 1D [<current|all>] [<single|multiple>] [type=<filetype>] [<filename>]

export 2D [type=<filetype>] [<filename>]

Options

Currently, the export command takes the following options:

fig[ure]|axis|axes|1D|2D
What to export.
"fig|figure" and "axis|axes" are all used to export the current axis, "1D" is used to export 1D datasets.
The optional additional arguments depend on that initial choice and are described below.
format (OPTIONAL)
Name of the figure format as defined in the fig2file.ini file.
Needs to be preceded by "format=".
filetype (OPTIONAL)
Figure export: File type the figure should be saved to, such as "pdf" or "fig".
1D/2D data export: File type the data should be exported to, such as "ASCII" or "glotaran".
Needs to be preceded by "type=".
caption option (OPTIONAL)
Figure export: Whether to save a caption together with the figure (and for that purpose display the caption GUI).
Needs to be preceded by "caption=".
filename (OPTIONAL)
Name of the file the figure or data should be saved to.
If no filename is provided, the user gets asked for a filename using the standard file selection dialogue boxes.
current|all (OPTIONAL)
Whether to export the currently active trace or all traces displayed.
Default is to use the option set in the GUI. If the option is provided, it will override what is set in the GUI.
single|multiple (OPTIONAL)
When exporting multiple traces, whether to save them all together in one file or each in a separate file.
Default is to use the option set in the GUI. If the option is provided, it will override what is set in the GUI.
