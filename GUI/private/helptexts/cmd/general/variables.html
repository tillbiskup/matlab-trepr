Variables

Often, particularly in context of scripting, using variables is quite useful, if not necessary. Therefore, variables have been introduced with version 0.3.37 of the toolbox.

If in a hurry

Variables can be set using the command define and used by typing the variable name with "$" as a prefix. There exists as well a set of predefined variables with special meaning. The feature is currently still EXPERIMENTAL.

Defining variables

Variables can be set using the command define. Just to give an example:

define foo = bar

For further information see the respective command reference page.

Using variables

Variables can be used as options in every command. They get expanded prior to parsing the actual command.

To use a variable, just prepend its name with the $ sign.

Predefined variables

There is a (growing) number of predefined variables for special tasks. Following a list of these special (predefined) variables and their meaning:

current
Index of the currently active dataset.
date
Date the currently active dataset has been recorded.
label
Label of the currently active dataset.
If you append #<index>, the label of the dataset corresponding to index will be returned. For details, see below.
pwd
Current directory (on the Matlab command line).
(The command is issued in the Matlab "base" workspace.)
ndatasets|numberofdatasets
Number of currently loaded datasets (sum of invisible and visible)
today
Current date.

Special case: dataset-specific variables

Guess you want to average parts of a dataset and reuse its label for the new dataset (just appending additional information). As long as you work on the currently active dataset, this can be done using just $label.

But what if you want to refer to a label of a specific dataset? All you need to know is the internal index of the dataset, as (now) displayed in the listboxes of datasets. To get the label of, let's say, the second dataset (if there is such thing as a second dataset), just type $label#2. Here, it doesn't matter whether you prepend the index with zeros, $label#002 will do the same for you.

Currently (version 0.3.49) this works only with label and date, but it will most probably be expanded as soon as there is some need for other variables as well.
