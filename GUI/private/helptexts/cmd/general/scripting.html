Scripting

The command line (CMD), once properly implemented, provides the user with the possibility to control most parts of the GUI. This immediately leads to the idea of scripting: performing tasks by running a set of commands saved in a file.

If in a hurry

Scripting allows you to perform a set of commands by loading a script rather than by typing the commands one by one into the command line. The feature is currently still EXPERIMENTAL.

Running scripts

Currently, the only way to run a script is from the "internal" panel of the GUI or by pressing the button labelled with "E" next to the command line. Thus you can load a script file and execute it. In the future, there will appear a run command in the command line for this.

Scripting: A few ideas

At the time of writing this text, scripting is still in its infancy. Therefore, this help serves currently as a reminder of what would be nice to have in the future.

Scripts for standard tasks

The power of scripting comes in very handy if certain tasks should be performed regularly on a dataset. Here, a well-written (and documented) script can save a lot of time.

Variables in commands

The situation: Many commands need specific options such as labels or file names. This makes it difficult to write generic scripts working with many different datasets.

The idea: Variables for fields in a dataset that get parsed and replaced with the respective values during evaluation of the command.

The (current) implementation: There is a new command define that lets you define variables within the CMD line. Variables can be used within commands by prepending the variable name with a "$".

Header of script files

Each script is just a collection of commands known by the CMD feature of the toolbox. Already now, comment lines are allowed. That would allow for using a header for each script file making it easier to get an idea what the script was all about, what purpose it serves, etc.

Fields for such a header would be a short description, perhaps keywords, an author and a date - and perhaps even more fields. For an idea why this would be useful, see the next section.

GUI for flicking through and loading scripts

The idea: a GUI window similar to what exists already for loading snapshots: A listbox for selecting the script, and for each selected script a few additional pieces of information making it much easier to decide whether this is the correct one.
